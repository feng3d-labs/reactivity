技术分析

@feng3d/reactivity 核心优势

  • 脏标记机制：自下而上的状态维护，变化只冒泡一次到父节点
  • 最小化代理开销：直接访问 ref.value，无额外 Proxy 包装
  • 高效批处理：批处理使用栈管理，避免重复依赖通知
  • 精确依赖追踪：依赖收集直接存储在响应式对象中

@vue/reactivity 设计特点

  • 版本号机制：自上而下的状态维护，每次访问检查版本号
  • Proxy 全面包装：所有响应式对象都经过 Proxy 处理
  • 队列式批处理：批处理使用队列调度，需要额外管理开销
  • 双向依赖维护：依赖关系需要双向维护，内存占用较高

适用场景建议

  • @feng3d/reactivity：适合复杂嵌套场景、大规模数据更新、性能敏感应用
  • @vue/reactivity：适合 Vue 生态集成、中小型应用、与 Vue 组件深度配合

测试场景说明

  • 基础操作：ref、computed 基本操作的创建和访问性能
  • 数组操作：大数组的读写、迭代、增删操作性能
  • 嵌套对象：深层嵌套对象的访问和更新性能
  • effect 追踪：依赖收集和 effect 触发性能
  • 批量更新：批量更新多个响应式值的性能
  • Map/Set 集合：响应式集合操作的增删改查性能
  • 复杂情况取值：极端嵌套场景下的计算性能
